Digital Check-up: autoavaliação conversacional guiada por IA que coleta sinais vitais (manuais ou simulados via wearables), sintomas atuais e dispara rastreios 
preventivos com detecção de sinais de alerta (red flags clínicas).

Prescrição Remota Inteligente: agente de IA que, após a teleconsulta, sugere prescrições baseadas no histórico do paciente, valida interações medicamentosas e 
encaminha para aprovação do médico — nunca substituindo o profissional, apenas acelerando o fluxo.

Componentes técnicos esperados:

Chatbot conversacional com contexto clínico injetado via system prompt e memória de sessão.
RAG (Retrieval-Augmented Generation) sobre base de conhecimento clínico e políticas Care Plus.
Function Calling para consulta simulada a histórico do paciente, verificação de interações medicamentosas e agendamentos.
Arquitetura multi-agente (opcional, bônus) usando LangGraph para orquestrar check-up, triagem e prescrição.
Suite de avaliação automatizada (evals) para medir qualidade das respostas.

Observações Importantes

O BluaDiagnostics lida com contexto clínico. Nenhuma resposta do agente pode afirmar diagnóstico definitivo ou substituir médico — isso deve estar explícito no system prompt 
e validado por casos no eval set.

Nenhuma API key pode aparecer em commits. Utilizar variáveis de ambiente, python-dotenv ou Google Colab Secrets. Commits com chaves expostas terão desconto na nota.

A escolha do modelo deve ser justificada tecnicamente — "é o que eu já usei" não é justificativa. Critérios esperados: adequação clínica, suporte a function calling 
estruturado, custo e privacidade.
